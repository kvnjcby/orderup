SQL Schema

  • Table Foods
    • id (int primary key)
    • name (varchar)
    • description (varchar)
    • imageURL (varchar)
    • price (decimal(5,2))
    • cookTime (int)
  • Table Orders
    • id (int primary key)
    • customerName (varchar)
    • orderStatus (int)
  • Table OrderedFoods
    • id (int primary key)
    • foodID (int foreign key)
    • orderID (int foreign key)

RESTful API

  • /createFood POST Parameters: name, description, imageURL, price, cookTime. Adds food to database.
  • /getFood GET Parameters: id. Returns food matching id in JSON: {id, name, description, imageURL, price, cookTime}
  • /getFoods GET Parameters: N/A. Returns all foods in JSON format: [{id, name, description, imageURL, price, cookTime}, ...]
  • /deleteFood DELETE Parameters: id. Deletes food by id.
  • /createOrder POST Parameters: customerName, orderStatus, foods. Foods parameter is an array of food ids [id, id, ...]. Creates new order.
  • /getOrder GET Parameters: id. Returns single order in JSON: {id, customerName, orderStatus, [foodID, foodID, ...]}
  • /getOrders GET Parameters: (optional)orderStatus. Returns all orders in JSON: [{id, customerName, orderStatus, [foodID, foodID, ...]}, ...].
  • /updateOrder PUT Parameters: id, orderStatus. Updates the status of an order based on id.
  • /deleteOrder DELETE Parameters: id. Deletes an order by id.
